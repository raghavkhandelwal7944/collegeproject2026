A Multi-Layer Firewall for Secure Large Language Model Interactions

Raghav Khandelwal, Swarnalatha P

Department of Computer Science and Engineering

Vellore Institute of Technology, India

raghav.khandelwal2022@vit.ac.in, swarnalatha@vit.ac.in

Abstract—Large Language Models are increasingly used in conversational systems, but direct exposure of raw prompts to models creates risks such as prompt injection, privacy leakage, unsafe content generation, and repeated sensitive-query processing. This paper presents Firewall LLM, a full-stack security gateway for LLM applications built using FastAPI and Next.js. The system introduces a layered protection pipeline combining prompt risk inspection, personally identifiable information redaction, token vaulting, semantic caching, and authenticated access control. The backend integrates Gemini-based prompt screening, Redis-assisted privacy and caching services, MySQL and MongoDB persistence, and a React dashboard for interaction monitoring. The proposed system demonstrates a practical architecture for safer LLM deployment in local or hybrid inference environments.

I. Introduction

The rapid adoption of LLM-based assistants in education, software engineering, and enterprise workflows has created a need for reliable security boundaries around model interaction. Prompts submitted by users may contain sensitive personal data, malicious instructions, or attempts to bypass safeguards. Without an intermediate filtering layer, such requests may be forwarded directly to a model, resulting in privacy compromise or unsafe outputs.

This project addresses that problem by designing and implementing a secure LLM firewall that sits between the user interface and model-serving layer. The system is intended for controlled deployments where model safety, prompt hygiene, and system observability are important. The work combines backend enforcement, persistent storage, authentication, and a modern monitoring dashboard.

The key contributions of this work are:

  • a multi-layer prompt security workflow for LLM applications;
  • a practical integration of PII redaction, risk detection, token vaulting, and semantic caching;
  • a complete web-based implementation with authentication, analytics, and chat history support.
II. Methodology
A. System Architecture

The proposed architecture consists of a FastAPI backend, a Next.js 16 frontend, and supporting infrastructure services including MySQL, MongoDB, Redis, Ollama, and Gemini. Users interact through a web dashboard, while the backend processes requests through a layered firewall pipeline before forwarding content to the target LLM. The architecture separates user interaction, policy enforcement, data persistence, and model communication into modular components.

B. Security Workflow

The request lifecycle is organized as a staged pipeline. First, the application validates the user session using JWT-based authentication. Second, prompt content is examined for malicious or unsafe intent using rule-based and model-assisted inspection. Third, privacy-sensitive entities such as email addresses, phone numbers, and API keys are redacted or replaced. Fourth, Redis-backed token vault and semantic cache mechanisms improve privacy retention and avoid unnecessary repeated inference. Finally, the sanitized request is sent to the configured model endpoint.

  1. User authentication and session validation.
  2. Prompt risk analysis and injection detection.
  3. PII masking and token replacement.
  4. Semantic cache lookup for similar previous queries.
  5. LLM invocation and monitored response delivery.
C. Backend Implementation

The backend is implemented in Python using FastAPI. It includes authentication endpoints for user registration and token issuance, chat-processing routes, health endpoints, and policy services. During startup, the application initializes the database schema, loads the Presidio service, configures the shared HTTP inference client, loads the sentence-transformer embedding model, and connects to Redis when available. This startup lifecycle reduces cold-start delays during the first user request.

D. Frontend Design

The frontend is implemented using Next.js 16, React 19, TypeScript, and Tailwind CSS. It provides login, signup, dashboard, policy views, traffic monitoring, and a playground chat interface. The user interface also computes a visible prompt risk indicator to provide immediate feedback during interaction. State management and API integration are handled using lightweight utility modules and typed request helpers.

III. Implementation Details
A. Core Components

The implemented system contains several coordinated modules. The FastAPI backend exposes registration, token generation, chat, policy, and health endpoints. Database support combines MySQL for user and policy records with MongoDB for conversational history. Redis is used to support token vault and semantic cache features. The frontend dashboard exposes the operational state of the firewall through widgets and chat views.

B. Technology Stack

The project uses FastAPI, HTTPX, Redis asyncio, Python-Jose, Presidio, spaCy, Sentence Transformers, Gemini, and Ollama on the backend. The frontend uses Next.js, React, TypeScript, Tailwind CSS, Framer Motion, Recharts, and related development tooling. This combination supports both security analysis and an interactive web experience.

C. Validation Strategy

The project includes unit tests for the firewall logic, including malicious prompt detection and redaction of email addresses, phone numbers, and API keys. These tests verify both positive and negative cases and help ensure repeatable validation for core security behaviors. Additional manual validation was conducted using the web interface, startup scripts, and direct API calls to confirm end-to-end functionality.

D. Sample Functional Capabilities
  • user registration and login using secure password hashing;
  • chat request forwarding through a controlled API layer;
  • prompt inspection for injection and unsafe intent;
  • redaction of sensitive entities before downstream processing;
  • traffic, policy, and telemetry views in the frontend dashboard.
IV. Evaluation
A. Evaluation Metrics

The system was evaluated qualitatively using correctness of redaction, response flow stability, user authentication behavior, and interpretability of the risk score presented in the frontend. Functional consistency was also examined by testing startup readiness, API communication between the frontend and backend, and safe handling of malicious prompts.

B. Observations

The multi-layer design improves practical safety by distributing responsibility across independent controls rather than relying on a single classification step. PII redaction reduces direct exposure of user secrets, while semantic caching improves repeated-query efficiency. The dashboard contributes transparency by showing the interaction state, risk signal, and operational flow to the user.

V. Applications

The proposed system can be applied in educational assistants, enterprise copilots, internal knowledge tools, and prototype AI products that require an intermediate security gateway. It is especially useful in environments where the model endpoint should not directly receive raw user input without inspection or privacy filtering.

VI. Discussion

This implementation demonstrates that LLM security can be approached as a layered systems problem rather than a single moderation call. However, the present solution still depends on externally configured models and local infrastructure availability. Future work can extend the platform with richer policy authoring, detailed audit trails, role-based access control, and quantitative benchmarking across attack categories.

VII. Conclusion

In this paper, a multi-layer security framework for LLM applications was presented and implemented as a complete working web platform. The system integrates secure authentication, prompt inspection, privacy-preserving preprocessing, semantic reuse, and dashboard-based monitoring. The resulting architecture offers a practical foundation for safer LLM deployment and demonstrates how modern web and AI infrastructure can be combined into a structured security gateway.
